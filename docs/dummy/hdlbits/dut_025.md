DUT=025 多级加法器端口连线
-----------------------------

- 问题：`make run_hdlbits_test DUT=025` 报 `Unsupported port connection expression` / `Assign LHS is not a memoized signal`，实例输出端口无法回接到父级信号，导致 elaboration NYI。
- 根因：层次实例化时，输出端口一律新建 SSA value，并通过 LHS converter 回写连接表达式。像 `add16 inst0(..., .cout(w1))` 这类纯符号连接，本应直接绑定父图里的 `w1` 值，但因为已有定义操作，回写路径找不到 memo，触发 NYI。
- 方案：`createInstanceOperation` 对输出端口先尝试 `resolveConnectionValue`，若得到的父级 value 尚无 definingOp，直接作为实例 result 绑定；否则再新建占位值并走 LHS 写回。输入端保持 RHS 转换。
- 验证：`make run_hdlbits_test DUT=025` 通过；`ctest` 全量通过（emit-sv、elaborate-hierarchy 恢复正常）。***
