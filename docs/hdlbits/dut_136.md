DUT=136 动态位选写入支持
--------------------------

- 问题：`make run_hdlbits_test DUT=136` 在 elaboration 阶段报 `Element select index must be constant on assign LHS`，因为 `out[i] <= in;` 使用了变量位选。
- 根因：LHSConverter 只接受常量位选，动态索引无法生成 flatten 路径和切片。
- 方案：在顺序 always LHS 路径上检测非常量位选，构造掩码/移位合成写回：获取当前 shadow/base 值，mask 清零被写位后用 `buildShiftedBitValue` 把 RHS 按索引左移并 OR 合并；若处于 guard 下再用 mux 保留旧值。生成全宽 slice 写回 WriteBackMemo，覆盖 seq 非 memory 信号的动态单比特/位段写。
- 验证：`make run_hdlbits_test DUT=136` 通过，hdlbits 仿真 `tb_136` PASS（仍有原题的 latch 警告仅告警不阻塞）。
