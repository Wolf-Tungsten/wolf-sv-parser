DUT=026 generate 实例重名
---------------------------

- 问题：`make run_hdlbits_test DUT=026` Verilator 报 16 个 `add1 fa` 重复实例名，生成的 SV 中 generate 展开的实例都叫 `fa`。
- 根因：`createInstanceOperation` 把实例名记录为原始 `childInstance.name`，未带层次路径，generate/数组实例被展平成多个 op 但共享同一个 `instanceName` 属性，emit 阶段优先使用该属性写入 SV。
- 方案：`instanceName` 改用层次路径 `deriveSymbolPath`，并做 sanitize；若路径为空再回退原名或自动编号。黑盒实例同样使用层次路径。
- 验证：`make run_hdlbits_test DUT=026` 通过，生成的 SV 实例名唯一，仿真输出 `[TB] dut_026 passed`。***
