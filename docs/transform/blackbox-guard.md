# blackbox-guard

## 功能概述

`blackbox-guard` pass 为未解析的黑盒模块（blackbox modules）创建 stub 图，并将可转换的黑盒操作转换为实例操作。

## 详细说明

在硬件设计中，黑盒模块是指那些只有接口定义而没有内部实现的模块。该 pass 执行以下操作：

1. **扫描黑盒操作**：遍历所有图（Graph）中的黑盒操作（`kBlackbox`）
2. **创建 Stub 模块**：对于无法找到对应图的黑盒模块，根据其端口信息创建 stub 图
3. **转换黑盒为实例**：对于已存在对应图的黑盒模块，将黑盒操作转换为实例操作（`kInstance`）

## Stub 模块创建

对于每个未解析的黑盒模块，pass 会：

- 提取模块名称、输入端口名称、输出端口名称和双向端口名称
- 根据端口连接推断端口位宽、符号和类型
- 创建一个新的图（Graph）作为 stub 模块
- 在新图中创建相应的输入端口、输出端口和双向端口

## 使用场景

- 处理部分实现的设计，其中某些模块只有接口定义
- 为后续的综合或仿真流程提供完整的模块层次结构
- 确保黑盒模块的端口签名一致性

## 命令行使用

```bash
# 在 transform pipeline 中使用
wolvrix --pass=blackbox-guard input.sv
```

## 注意事项

- 如果同一模块名称的黑盒具有不同的端口签名，会发出警告
- 端口数量不匹配时会使用默认位宽（1位）作为 fallback
- 创建的 stub 模块会保留原始黑盒的端口名称和类型信息
